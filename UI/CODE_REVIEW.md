# 프론트엔드 코드 리뷰 및 개선 사항

## 🔴 중요 개선 사항

### 1. **Cart.jsx - Key 사용 문제**
**문제**: `cartItems.map((item, index) => ...)` 에서 `key={index}` 사용
- 리스트 순서가 변경되면 React가 잘못된 컴포넌트를 업데이트할 수 있음
- 고유한 ID를 사용해야 함

**개선 방안**:
```javascript
// 현재
cartItems.map((item, index) => (
  <div key={index} className="cart-item">

// 개선
cartItems.map((item) => (
  <div key={`${item.menuId}-${item.selectedOptions.map(o => o.optionId).join('-')}`} className="cart-item">
```

### 2. **이미지 로드 실패 처리 없음**
**문제**: `MenuCard.jsx`에서 이미지 로드 실패 시 대체 이미지 표시 없음
- 이미지 파일이 없거나 로드 실패 시 빈 화면 표시

**개선 방안**: 이미지 로드 실패 시 SVG 폴백 사용

### 3. **사용하지 않는 함수**
**문제**: `MenuCard.jsx`의 `calculatePrice()` 함수가 정의되어 있지만 사용되지 않음

**개선 방안**: 함수 제거 또는 실제 사용

### 4. **성능 최적화 부족**
**문제**: `AdminPage.jsx`에서 주문 정렬이 매 렌더링마다 실행됨
```javascript
orders={[...orders].sort((a, b) => new Date(b.orderDate) - new Date(a.orderDate))}
```

**개선 방안**: `useMemo`로 최적화

### 5. **에러 처리 부족**
**문제**: 
- 이미지 로드 실패 처리 없음
- 주문 생성 실패 시 처리 없음
- 재고 업데이트 실패 시 처리 없음

## 🟡 개선 권장 사항

### 6. **접근성(Accessibility) 개선**
- 버튼에 `aria-label` 추가
- 키보드 네비게이션 지원
- 스크린 리더 지원

### 7. **사용자 피드백 개선**
- `alert()` 대신 토스트 메시지 사용
- 로딩 상태 표시
- 성공/실패 피드백 개선

### 8. **데이터 검증**
- 주문 생성 시 장바구니 비어있는지 확인 (현재는 있음)
- 재고가 음수가 되지 않도록 확인 (현재는 있음)
- 주문 금액 계산 검증

### 9. **코드 중복**
- 날짜 포맷팅 함수가 여러 곳에서 사용될 수 있음 → 유틸 함수로 분리
- 가격 포맷팅도 유틸 함수로 분리 가능

### 10. **타입 안정성**
- JavaScript만 사용 중이지만, JSDoc 주석 추가로 타입 힌트 제공 가능

## 🟢 선택적 개선 사항

### 11. **로컬 스토리지 활용**
- 장바구니 상태를 로컬 스토리지에 저장하여 새로고침 시에도 유지
- 주문 내역도 로컬 스토리지에 저장

### 12. **에러 바운더리 추가**
- React Error Boundary로 예상치 못한 에러 처리

### 13. **로딩 상태**
- 이미지 로딩 중 스켈레톤 UI 표시
- 주문 처리 중 로딩 인디케이터

### 14. **반응형 디자인 개선**
- 더 작은 화면 크기 대응
- 터치 인터랙션 개선
